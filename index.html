Belajar Konfigurasi
Bersama Con-Fight
2025

Con-Fight adalah platform pembelajaran yang dirancang untuk membantu siswa menguasai materi melalui latihan, tantangan, dan ulasan materi yang terstruktur. Platform ini memadukan Learning Management System sebagai wadah materi dan instruktur khusus untuk membimbing proses konfigurasi. Sedangkan proses konfigurasi tersedia dalam bentuk Virtual Machine yang sudah di automatisasi sehingga dapat diakses secara remot. Kami percaya bahwa proses belajar akan lebih efektif jika dibarengi dengan semangat kompetisi dan keterlibatan aktif.

Tentang Kami Materi

Tentang Kami

Tim di balik Con-Fight merupakan dari pendidik yang berkomitmen menghadirkan pengalaman belajar digital yang menyenangkan, inklusif, dan efektif. Kami percaya bahwa teknologi harus memperkuat proses belajar, bukan menggantikannya. Dengan Con-Fight, kami ingin menciptakan ruang belajar modern yang mendukung siswa berkembang sesuai kecepatan dan gaya mereka masing-masing.

Tujuan

Terdapat 3 tujuan pembelajaran yang akan dicapai yakni:

Memahami Konsep Web-Server & DNS Server

Peserta didik mampu memahami konsep dan fungsi dari layanan Web Server dan DNS Server beserta perintah dasar yang digunakan.

Memahami Konfigurasi Web Server & DNS Server

Peserta didik mampu memahami proses konfigurasi dari layanan Web Server dan DNS Server serta mengintegrasikan keduanya.

Mampu Menganalisis & Memecahkan Masalah Keduanya

Peserta didik mampu menganalisis dan memecahkan permasalahan dari Web Server dan DNS Server serta menemukan solusinya.

Capaian Pembelajaran

Peserta didik dapat memahami konsep, menerapkan, mendokumentasikan, mengomunikasikan, serta memecahkan masalah secara prosedural terkait layanan dan konfigurasi Web Server, Database Server, DNS Server, Mail Server, Directory Server, teknologi kontainer, menerapkan manajemen platform (backup dan recovery).

FAQ

Untuk membantumu memahami lebih dalam tentang platform Con-Fight, kami telah mengumpulkan beberapa pertanyaan yang sering ditanyakan oleh pengguna. Jika kamu tidak menemukan jawaban yang kamu cari, jangan ragu untuk menghubungi tim kami melalui halaman Contact Us.

Untuk mengakses Virtual Machine (VM), silahkan anda meminta akses kepada guru anda dan langsung mengerjakan praktikum yang diberikan. Contoh akses yang akan diberikan ialah ssh -p [port] user@10.10.10.10. Jika ada kendala, jangan ragu untuk bertanya ke guru anda!

Berikut perintah yang anda bisa gunakan ketika praktikum:
lab start [namaExercise] digunakan untuk memulai praktikum dan mengisolasi prosesnya
lab grade [namaExercise] digunakan untuk menilai hasil pengerjaan praktikum, anda bisa melakukan pengulangan konfigurasi sampai memenuhi nilai
lab finish [namaExercise] digunakan untuk mengunci hasil nilai dan mengirimkannya ke guru anda

Saat ini, materi mencakup:
-Pengenalan Web Server & DNS Server
-Konfigurasi dasar dan lanjutan
-Latihan interaktif & simulasi troubleshooting

Anda tidak perlu menginstall VM! Anda hanya perlu mengaksesnya secara remote. Bisa melalui CMD, PUTTY, atau Aplikasi lainnya yang mendukung protocol SSH.
